Workshop

The goal of this workshop is to show you how you can setup a CI/CD pipeline. To make it rather simple, this exact application which you're now visting will be used for this purpose.

The application is a mini-website (built using the Flask micro-framework) programmed in Python. It has only about 200 lines of Python code (excluding tests).

Due to time constraints, a limited pipeline will be set up. However, the basic principles of the services will be demonstrated, which should allow you to set up a complete pipeline if interested.

Used tools

We will focus on using "free" (for open-source) tools that can be (easily) integrated into GitHub so that you don't need to bother with setting up local instances.

GitHub

GitHub is a DVC (Distributed Version Control) service which offers SCM (Source Code Management) using Git. GitHub offers unlimited public repositories for free, and paid private repositories.

Some alternatives for GitHub are BitBucket and GitLab. For this workshop GitHub was chosen because it has the largest collection of integrations available.

Travis CI

Travis CI is the service that is doing the heavy-lifting. It automatically gets notified of the changes you pushed to GitHub, and then pulls in those changes and performs a set of tasks for you.

Most of the time this includes creating builds (with multiple OS'es, or programming language versions), running tests and also deploying the code.

Alternatives for Travis CI include CircleCI, AppVeyor and Semaphore.

Codecov

Codecov is a code quality measurement tool. It generated coverage reports based on the data it receives from test suites that are ran with coverage.

This enables you to check on how much of your code is covered by unit tests.

The reporting step in this workshop has been worked out with Codecov, but Coveralls or CodeClimate could be used just as well.

Heroku

Heroku offers a PaaS (Platform as a Service), which can be used to run the application on without having to worry about the back-end or underlying platform.

We will use Heroku to deploy this app onto automatically once the tests pass.
